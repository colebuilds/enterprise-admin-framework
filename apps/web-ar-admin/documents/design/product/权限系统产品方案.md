# 权限系统产品方案

## 1. 背景

AR 平台管理后台面向多个角色（超管、运营、财务、商户管理员等），不同角色只能访问其职责范围内的页面和操作。权限体系保证：

- 没有权限的人看不到对应的菜单入口
- 进入了页面但没有操作权限，对应按钮不显示
- 前端和后端双重拦截，前端权限控制是防误操作，后端是最终防线

## 2. 权限控制粒度

### 2.1 页面级（能不能进入这个页面）

用户只能访问其权限码覆盖的页面。无权限时：

- 导航菜单不显示该入口
- 直接访问 URL 跳转至 403 页面

### 2.2 操作级（页面内能不能做这个操作）

同一页面内，不同操作对应不同权限码。常见操作：

- 新增、编辑、删除
- 导出、批量操作
- 审核、转派、强制操作等高风险动作

无权限时，按钮从 DOM 中移除（默认），或保留但置为禁用状态（产品明确要求时）。

### 2.3 数据级（只看自己的数据）

数据范围控制（如商户只看自己的数据）由后端接口层保证，前端不承担数据过滤责任。

## 3. 权限码的来源

权限码由后端在角色管理中配置，通过登录后获取用户信息接口返回。

权限码命名示例：

| 层级 | 示例                                                         |
| ---- | ------------------------------------------------------------ |
| 2 层 | `SchedulerTask:Create`                                       |
| 3 层 | `SystemManage:SysUserPage:Add`                               |
| 4 层 | `SystemManage:SysUserPage:UserPage:Add`                      |
| 5 层 | `Finance:WithdrawOrderManage:Dashboard:Withdraw:GetPageList` |

层级越深，粒度越细。同一页面通常有一个页面入口码 + 若干操作码。

## 4. 无权限时的产品行为

| 场景                     | 行为                           |
| ------------------------ | ------------------------------ |
| 菜单入口                 | 不显示（菜单树中不出现）       |
| 直接访问受控 URL         | 跳转 403 页面                  |
| 页面内按钮/操作          | 从 DOM 移除（不可见，不占位）  |
| 需要"可见但不可点"的场景 | 按钮保留但禁用，需产品明确说明 |
| 本地开发环境             | 自动开放全部权限（无需配置）   |

## 5. 与字典、i18n 的关系

权限系统只控制"能不能做"，不影响字典数据加载和多语言显示。字典和多语言在登录后统一加载，权限过滤在路由和组件层独立运行。
