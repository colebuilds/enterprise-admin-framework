# i18n 产品架构

## 1. 文档定位

本文描述 `apps/web-ar-admin` 当前使用的国际化系统整体架构。

回答的核心问题：

- 这套系统由哪几层组成，每层的职责边界在哪
- 一次语言切换，从用户点击到页面文字更新，经历了哪些环节
- locale 状态存在哪、谁读、谁写
- NaiveUI 组件库的 locale 是如何联动的

执行细节（接入、用法、key 命名）见： `documents/guides/development/国际化接入与运行时机制说明.md`

技术实现说明（vben 对 vue-i18n 做了什么改造、为什么不直接用 vue-i18n）见： `documents/design/technical/2026-05-08-i18n-技术实现.md`

## 2. 三层架构

```
┌─────────────────────────────────────────────────────────┐
│  Layer 3 — App Layer (apps/web-ar-admin)                │
│                                                         │
│  src/locales/index.ts        ← 应用入口，注册 loadMessages 钩子
│  src/locales/langs/          ← 业务翻译 JSON 资源        │
│  src/app.vue (NAIVE_LOCALE_MAP) ← NaiveUI locale 联动   │
│  @vben/preferences            ← locale 持久化存储        │
└───────────────────┬─────────────────────────────────────┘
                    │ coreSetup(app, { loadMessages })
┌───────────────────▼─────────────────────────────────────┐
│  Layer 2 — @vben/locales (packages/locales)             │
│                                                         │
│  setupI18n()         ← 安装 i18n 到 Vue app             │
│  loadLocaleMessages() ← 加载语言包（幂等）               │
│  loadLocalesMapFromDir() ← glob 扫描目录，构造 locale 加载器 │
│  setI18nLanguage()   ← 更新 i18n.global.locale + html lang │
│  useSimpleLocale()   ← 轻量 UI Kit locale（不依赖 vue-i18n）│
└───────────────────┬─────────────────────────────────────┘
                    │ createI18n
┌───────────────────▼─────────────────────────────────────┐
│  Layer 1 — vue-i18n (third-party library)               │
│                                                         │
│  i18n 单例（legacy: false, Composition API 模式）         │
│  i18n.global.t(key)  ← 翻译函数                          │
│  i18n.global.locale  ← 当前 locale Ref                   │
│  setLocaleMessage / mergeLocaleMessage ← 消息注入         │
└─────────────────────────────────────────────────────────┘
```

## 3. 支持的语言

| 代码（BCP47） | 中文名   | NaiveUI 内置 locale |
| ------------- | -------- | ------------------- |
| `zh-CN`       | 简体中文 | `zhCN` / `dateZhCN` |
| `en-US`       | English  | `enUS` / `dateEnUS` |

默认语言：`zh-CN`（由 `@vben/preferences` 配置初始值决定）

当前只支持这两种语言，与 vben 核心包的 `SupportedLanguagesType = 'en-US' | 'zh-CN'` 完全对齐，无需 `as any` 绕过。`app.vue` 中 `NAIVE_LOCALE_MAP` 保留了 vi-VN/id-ID 条目（无害），待将来产品决策支持时直接扩展。

## 4. 翻译资源分布

### 4.1 vben 框架自带资源

位于 `packages/locales/src/langs/`，仅提供 `zh-CN` / `en-US` 两套：

```
langs/
  en-US/
    authentication.json   ← 登录/注销页文案
    common.json           ← 通用词汇
    preferences.json      ← 偏好设置面板文案
    profile.json          ← 个人资料页文案
    ui.json               ← UI Kit 组件文案
  zh-CN/
    (同上，5 个文件)
```

这些资源由 `loadLocalesMapFromDir` 扫描，在 `setupI18n` 阶段合并进 i18n 消息树。

### 4.2 业务应用资源（本项目）

位于 `src/locales/langs/`，按语言目录 + 业务文件组织：

```
langs/
  zh-CN/
    page.json       ← 页面级通用文案
    menu.json       ← 菜单名称
    (其他业务模块...)
  en-US/
    (同上)
  vi-VN/
    (同上)
  id-ID/
    (同上)
```

通过 `src/locales/index.ts` 注册的 `loadMessages` 钩子，在每次 `loadLocaleMessages` 调用时注入到 i18n 消息树。

### 4.3 消息树结构

两层资源合并后，消息树的结构为：

```
locale: 'zh-CN'
messages: {
  // vben 框架层 (直接展开到根)
  'authentication.signIn': '登录',
  'common.cancel': '取消',
  ...

  // app 层 (以文件名为 namespace 前缀)
  page: { ... },
  menu: { ... },
}
```

## 5. Locale 状态流转

### 5.1 初始化（应用启动）

```
main.ts
  → setupI18n(app)                         [src/locales/index.ts]
    → preferences.app.locale               [读 localStorage]
    → coreSetup(app, { defaultLocale, loadMessages })
      → app.use(i18n)                      [安装 vue-i18n]
      → loadLocaleMessages(defaultLocale)
        → loadLocalesMapFromDir 合并 vben 资源
        → loadMessages(locale) 合并 app 资源
        → setI18nLanguage(locale)          [更新 i18n.global.locale + <html lang>]
```

### 5.2 语言切换（用户操作）

```
LanguageToggle.vue 或 PreferencesDrawer.vue
  → updatePreferences({ app: { locale } })  [写入 preferences + localStorage]
  → loadLocaleMessages(locale)              [@vben/locales]
    → setSimpleLocale(locale)              [更新 useSimpleLocale 的轻量 locale]
    → 合并 vben 框架资源（vben 层）
    → loadMessages(locale)                 [合并 app 业务资源]
    → setI18nLanguage(locale)              [i18n.global.locale.value = locale]
                                           [document.html.lang = locale]

app.vue (useLocaleSync — 监听 preferences.app.locale 变化)
  → invalidateDicts()                      [标记所有字典 cache stale，触发 background refetch]
  → api.system.changeLanguage()            [已登录时同步后端语言偏好，fire-and-forget]

app.vue (NAIVE_LOCALE_MAP — 响应式)
  → computed(() => NAIVE_LOCALE_MAP[preferences.app.locale])
  → NaiveUI locale 自动更新
```

### 5.3 locale 副作用：useLocaleSync

`useLocaleSync` 是 app 层的一个全局单例 composable，在 `App.vue` 的 setup 中调用一次。

它 watch `preferences.app.locale`，在每次语言切换时触发两个副作用：

| 副作用 | 函数 | 条件 |
| --- | --- | --- |
| 字典缓存失效 | `invalidateDicts()` | 无条件（切语言即失效） |
| 后端语言同步 | `api.system.changeLanguage()` | 已登录（`accessStore.accessToken` 存在） |

字典失效后，TanStack Query 对 active 组件中的 `useQuery` 自动触发 background refetch，用户不感知加载过程。

### 5.4 后续请求

切换语言后，`requestClient` 在 `signBody` 中调用 `resolveRequestLanguage()` 读取 `preferences.app.locale` 并转换为后端短码（`zh-CN → zh`）。无需额外事件订阅，每次请求时实时读取。

## 6. locale 状态的存储位置

| 状态 | 存储位置 | 读取者 | 写入者 |
| --- | --- | --- | --- |
| 运行时 locale | `i18n.global.locale`（Ref） | `$t()`, `useI18n()` | `setI18nLanguage()` |
| 应用偏好 locale | `preferences.app.locale`（响应式对象） | `app.vue`, `language-toggle.vue` | `updatePreferences()` |
| 持久化 locale | `localStorage['preferences-locale']` | `preferences` 初始化时读取 | `preferences.ts` 自动同步 |
| 轻量 UI locale | `useSimpleLocale().currentLocale`（Ref） | UI Kit 弹框、按钮 | `setSimpleLocale()` |

这四处状态在正常流转下始终同步，因为所有写入路径都经过 `loadLocaleMessages`。

## 7. NaiveUI 集成

`app.vue` 维护一个静态映射表将 vben locale 代码转换为 NaiveUI 内置 locale 对象：

```ts
const NAIVE_LOCALE_MAP = {
  'zh-CN': { locale: zhCN, dateLocale: dateZhCN },
  'en-US': { locale: enUS, dateLocale: dateEnUS },
  'vi-VN': { locale: viVN, dateLocale: dateViVN },
  'id-ID': { locale: idID, dateLocale: dateIdID },
};
```

`<NConfigProvider :locale="tokenLocale" :date-locale="tokenDateLocale">` 接收 computed 值，`preferences.app.locale` 变化时 NaiveUI 自动更新。

## 8. 与老项目方案的关键差异

| 维度 | 老项目 | 新项目（本文档） |
| --- | --- | --- |
| locale 代码格式 | 短码：`zh` / `en` / `vi` / `id` | BCP47：`zh-CN` / `en-US` / `vi-VN` / `id-ID` |
| 加载粒度 | namespace 级懒加载（路由进入时按需） | locale 级全量加载（首次切换时全部合并） |
| 预热策略 | 启动时只预热 `common + menu + components` | 启动时加载完整 locale（文件体积小，可接受） |
| 切语言行为 | 重放"已使用 namespace"集合 | 直接重新调用 `loadLocaleMessages`（全量） |
| locale 持久化 | 手动操作 localStorage | 由 `@vben/preferences` 统一管理 |
| NaiveUI 联动 | 无（老项目用 Element Plus） | `app.vue` NAIVE_LOCALE_MAP 响应式联动 |

## 9. 当前已知限制

- `document.title` 只在路由 `afterEach` 更新，留在当前页切语言不会立即更新标题
- vben 的 `SUPPORT_LANGUAGES` 常量只包含 2 个语言，如需在 UI 中展示 vi-VN/id-ID 选项，需在 app 层独立维护选项列表
- 切语言后 `invalidateDicts()` 只标记 stale，不主动触发请求；只有 active 组件中的 `useQuery` 才会 background refetch。非活跃页面的字典待用户访问时才重拉
