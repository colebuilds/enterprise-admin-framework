# 变更控制协议-极简执行版

## 1. 目标

把 AI 变更过程控制在“可确认、可审计、可回滚”的范围内。

## 2. 六步执行流

1. 任务理解（目标/不包含项）
2. 边界声明（允许改/禁止改）
3. 执行计划（文件+命令+风险）
4. 获得明确执行授权
5. 最小必要改动
6. 结果回报（含偏差审计）

## 3. 必守规则

- 未获得明确授权前，不落盘修改。
- 默认只做最小必要改动，不顺手扩改。
- 高风险动作（依赖、lockfile、批量变更）先暂停确认。
- 计划外变更必须说明原因并回报。

## 4. 必停条件

- 需要改计划外文件
- 需要新增/删除/重命名且未授权
- 需要安装依赖或改 lockfile
- 原方案无法完成目标

## 5. 执行后最小回报

- 实际修改文件 + 摘要 + 是否计划内
- 实际执行命令 + 结果 + 是否计划内
- 新增/删除/重命名列表（无则写无）
- 偏差审计：计划外改动/命令/未完成项/禁止范围触碰
