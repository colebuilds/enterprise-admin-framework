# 子任务协作规范-极简执行版

## 1. 何时拆分

满足任意两项可拆：

- 多模块/多目录
- 可并行盘点
- 分析与实施并存
- 主代理持续卡住

## 2. 何时不拆

- 小任务（1~2 文件）
- 高耦合单链路
- 边界不清
- 会频繁改同一核心文件

## 3. 角色边界

- 主代理：收边界、分配所有权、统一收口。
- 子代理：只在授权边界内执行，发现冲突立即回主代理。

## 4. 执行顺序

1. 主代理收边界
2. 并发只读盘点
3. 主代理定方案
4. 并发实施
5. 主代理统一验收与汇报

## 5. 风险控制

- 同一核心文件默认单人负责。
- 入口/索引/全局配置默认主代理负责。
- 未获明确执行授权前，不落盘修改。
