# 1. 目的

建立项目级统一规则，约束 `src/api/` 目录下接口定义、类型定义与导出文件的生成和维护方式，避免人工直接修改导致接口资产不一致、类型漂移、生成结果与源码脱节等问题。

# 2. 适用范围

- `src/api/` 目录下的所有接口模块
- 接口定义文件（如 `index.ts`）
- 接口类型定义文件（如 `types.ts`）
- 与 OpenAPI 生成链路直接相关的接口资产

# 3. 核心规则

1. `src/api/**/*` 默认视为生成资产，不作为人工直接维护目录。
2. 禁止直接创建、修改、删除 `src/api/` 下的接口定义文件和类型定义文件。
3. 命中接口变更、接口新增、接口删除、字段变化、DTO 变化时，必须优先通过标准生成流程处理，而不是手工修补生成结果。
4. 任何绕过生成流程的人工修改，都应视为例外处理，并需要明确说明原因与后续回收方式。

# 4. 标准做法

## 4.1 生成入口

统一使用项目标准命令生成和更新接口资产：

```bash
npm run gen:api
```

如果项目后续统一切换到其他等价命令，应同步更新本规范与项目入口文档。

## 4.2 正确流程

当任务涉及接口层改动时，默认按以下顺序执行：

1. 先确认是否真的命中 `src/api/` 资产变更
2. 如果命中，优先运行 `npm run gen:api`
3. 检查生成结果是否符合当前接口契约
4. 再在页面层、组合层、数据映射层继续后续实现

## 4.3 生成后最小检查项

- 生成命令成功执行
- 相关接口文件与类型文件已更新到预期状态
- 页面侧引用没有继续依赖过期字段或错误类型
- 未引入人工手改 `src/api/` 的残留

# 5. AI 协作边界

1. AI 不得直接编辑 `src/api/**/*` 下的文件，除非用户明确授权处理例外场景。
2. 当任务涉及接口定义或 DTO 变化时，AI 应明确指出该任务命中 API 生成链路，并优先建议或执行 `npm run gen:api`。
3. AI 可以：
   - 分析接口变更影响
   - 执行生成命令
   - 检查生成结果
   - 修改 `src/api/` 之外的使用方代码
4. AI 不可以：
   - 将页面侧临时需求直接手工补进生成文件
   - 为了“快速通过编译”在 `src/api/` 中做一次性热修
   - 把生成资产当作普通业务源码长期维护

# 6. 例外处理

以下情况才考虑例外：

- 生成工具本身存在故障，且当前任务被明确要求临时阻断外处理
- 上游 OpenAPI 描述明显错误，必须先做短期兼容
- 项目处于迁移阶段，存在经确认的过渡性维护窗口

发生例外时，必须至少满足：

- 明确记录为什么不能走标准生成流程
- 明确说明临时修改范围
- 明确说明后续何时回归标准生成链路

# 7. 验收标准

- 接口资产变更通过标准生成命令完成
- `src/api/` 未出现未经说明的人工直接修改
- 使用方代码已与最新生成结果对齐
- AI 与人工协作过程对该边界保持一致理解
