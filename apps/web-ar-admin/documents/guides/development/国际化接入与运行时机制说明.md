# 国际化接入与运行时机制说明

## 1. 文档定位

本文是 AR admin 前端国际化的**开发者操作手册**，面向需要接入多语言的功能开发者。

回答的实际问题：

- 怎么在模板和脚本里翻译文字
- 怎么添加新的翻译 key
- 怎么给新模块接入翻译文件
- 语言切换时会发生什么，需要关注什么
- 发请求时怎么带语言参数

方案设计和技术原理见：
`documents/design/technical/2026-05-08-i18n-产品架构.md`
`documents/design/technical/2026-05-08-i18n-技术实现.md`

## 2. 快速参考

### 2.1 在模板中翻译

```vue
<template>
  <span>{{ $t('page.title') }}</span>
  <NButton>{{ $t('common.confirm') }}</NButton>
</template>
```

### 2.2 在 `<script setup>` 中翻译

```ts
import { useI18n } from 'vue-i18n';

const { t } = useI18n();

const message = t('page.saveSuccess');
const label = computed(() => t('page.filterLabel'));
```

### 2.3 在非组件上下文（工具函数、store）中翻译

```ts
import { $t } from '#/locales';

// $t 是 i18n.global.t 的引用，可在任何地方使用
const message = $t('common.networkError');
```

## 3. 翻译文件组织

### 3.1 目录结构

```
src/locales/langs/
  zh-CN/
    page.json        ← 通用页面文案
    menu.json        ← 菜单名称
    finance.json     ← 财务模块文案（按业务域拆分）
    system.json
    ...
  en-US/
    (与 zh-CN 文件集对齐)
```

当前只支持 `zh-CN` 和 `en-US`。每新增一个 JSON 文件，须**两套语言目录同步新增**，否则切语言时该文件对应的翻译丢失。

### 3.2 文件内 key 结构

同一文件内的 key 建议使用嵌套对象表达层次关系：

```json
// finance.json
{
  "finance": {
    "withdrawOrder": {
      "title": "提现工单",
      "filter": {
        "status": "状态",
        "dateRange": "时间范围"
      },
      "table": {
        "orderId": "工单号",
        "amount": "金额"
      }
    }
  }
}
```

调用：`$t('finance.withdrawOrder.filter.status')`

### 3.3 通用词汇使用 page 或 common 命名空间

按钮、操作、状态这类跨业务的通用词汇放在 `page.json`（或已有 `common.json`）：

```json
// page.json
{
  "actions": {
    "add": "新增",
    "edit": "编辑",
    "delete": "删除",
    "export": "导出"
  },
  "status": {
    "enabled": "启用",
    "disabled": "禁用"
  }
}
```

**不要把通用词汇散放到业务 JSON 里**，会造成翻译不一致。

## 4. 给新模块添加翻译

以新增"报表"模块为例：

**Step 1：在两个 locale 目录下同时创建 JSON 文件**

```
src/locales/langs/zh-CN/report.json
src/locales/langs/en-US/report.json
```

**Step 2：编写中文版（以 zh-CN 为基线）**

```json
{
  "report": {
    "title": "报表中心",
    "rechargeReport": {
      "title": "充值报表",
      "total": "充值总额"
    }
  }
}
```

**Step 3：翻译 en-US 版本**

确保 key 结构和 zh-CN 完全一致，只有 value 不同。

**Step 4：在组件中使用**

```vue
<template>
  <h1>{{ $t('report.title') }}</h1>
  <h2>{{ $t('report.rechargeReport.title') }}</h2>
</template>
```

不需要任何 import 或注册操作。`loadLocalesMapFromDir` 会自动扫描 `src/locales/langs/` 下所有 JSON 文件。

## 5. 字典 label vs 本地翻译

这是最容易混淆的边界。

| 内容类型 | 来源 | 用法 |
|---|---|---|
| 页面标题、按钮文字、表头、提示语 | 本地 locale JSON | `$t('...')` |
| 状态枚举 label（启用/禁用、订单状态）| 后端字典 | `useDictOptions` / `useLabelMap` |
| 下拉选项（商户列表、角色列表）| 后端字典 | `useDictOptions` |
| 国家、语言、货币名称 | 后端字典 platform | `useDictOptions('countryList', 'platform')` |

**判断规则：** 这个文字是固定的 UI 标签吗 → 本地翻译。这个文字的值由后端数据决定吗 → 字典。

**反例（错误做法）：**
```ts
// ❌ 在 locale JSON 里维护状态枚举
// zh-CN/page.json
{ "status": { "0": "禁用", "1": "启用" } }

// ✅ 正确：走字典
const statusOptions = useDictOptions('enableStateList')
```

## 6. 语言切换的运行时行为

### 6.1 切换触发路径

语言切换只有两个入口（都在 vben layout 层）：

1. `LanguageToggle.vue` — 顶部导航栏的语言图标
2. `PreferencesDrawer.vue` — 偏好设置抽屉

两个入口都会：
- 调用 `updatePreferences({ app: { locale } })` 写入 preferences（同步持久化到 localStorage）
- 调用 `loadLocaleMessages(locale)` 异步加载并切换语言

### 6.2 切换后前端文案立即更新

本地 locale JSON 中的翻译（所有 `$t(...)` 调用）会在切换完成后立即响应，无需刷新页面。

### 6.3 切换后字典自动失效并重拉

`useLocaleSync`（在 `App.vue` 中全局注册）监听语言切换，自动调用 `invalidateDicts()`，将所有字典缓存标记为 stale。

- 当前页面中 active 的 `useDictOptions` / `useLabelMap` 会 background refetch，用户无感知
- 非活跃页面的字典等访问时再拉，不浪费

业务代码**不需要手动失效字典**，切语言后字典会自动跟随更新。

### 6.4 切换后同步后端语言偏好

`useLocaleSync` 在用户已登录时，会 fire-and-forget 调用 `api.system.changeLanguage()`，通知后端更新该用户的语言偏好（下次登录默认语言）。

业务代码无需关心，切语言即自动触发。

### 6.5 页面 title 可能不立即更新

`document.title` 在路由 `afterEach` 中更新。若用户留在当前页切语言而不跳路由，title 不会立即更新。

## 7. 请求层的语言参数

`requestClient` 的 `signBody` 函数在每次请求时调用 `resolveRequestLanguage()` 实时读取当前 locale 并转换为后端期望的格式：

- 前端 locale 码：`zh-CN` / `en-US`
- 后端期望短码：`zh` / `en`（与老项目保持一致）
- 转换规则：`locale.split('-')[0] ?? 'zh'`

语言切换后，下一个请求自动携带新语言码，业务代码无需关心。

如果某个接口需要手动读取当前语言短码：

```ts
import { preferences } from '@vben/preferences';

const lang = preferences.app.locale.split('-')[0] ?? 'zh'; // 'zh' | 'en'
```

## 8. 常见问题

### Q: 翻译 key 返回 key 本身（未找到翻译）

原因：
1. JSON 文件未创建（检查 4 个语言目录是否都有对应文件）
2. key 路径拼写错误
3. 文件内嵌套结构与调用路径不匹配

调试：开发环境下 `missingWarn: true` 已开启，控制台会打印 `[intlify] Not found '...' key` 警告。

### Q: 添加了 JSON 文件，但 $t 找不到这个文件的 key

确认 JSON 文件存放在 `src/locales/langs/<locale>/` 下（不是其他目录）。`loadLocalesMapFromDir` 只扫描这个固定目录。

### Q: 切语言后某些 NaiveUI 组件（DatePicker / Select）没有更新

确认 `app.vue` 的 `NAIVE_LOCALE_MAP` 包含了目标 locale，且 `<NConfigProvider :locale="tokenLocale">` 正确注入。如果是新增的 locale，需要在这里补充映射。

### Q: 写了新的 common 文案，发现已有同义词

优先复用已有 key，不为同一语义制造多套别名。如果现有 key 的语义确实不同，使用新的 key 并在 PR review 时说明区别。
