# 模块内多语言接入 AI 提示词模板

## 1. 目标

本文档用于提供一套可直接复制给 AI 的提示词模板，减少“为某个模块接入既有 i18n 方案”时反复补充上下文的成本。

它不是新的多语言方案说明，也不负责定义全局 i18n 机制；它只用于把一个明确模块接到仓库现有方案里。

如果你现在还没决定“该点哪个 skill、该怎么开口”，先看：

- `documents/guides/development/AI技能调用一页口令卡.md`

适用目标：

- 为某个明确模块或页面接入既有 i18n 方案
- 已有模块补齐漏掉的 locale
- 盘点某个模块里不支持 `switchLocale()` 的接入点

不适用目标：

- 设计新的多语言架构或全局治理方案
- 重写仓库现有的 i18n 运行时机制

当前运行时启用语种以 `src/i18n/index.ts` 的 `SUPPORT_LOCALES` 为准，现状为 `en`、`zh`、`vi`、`id`。默认至少补齐 `zh/en`；若目标 namespace 已存在 `vi/id` 资源，同步补齐。

---

## 2. 默认使用方式

推荐直接把下面模板中的 `[模块/页面路径]` 替换成真实目标后发送给 AI。

优先填写：

- 页面目录路径，例如 `src/views/operations/takePackage`
- 或页面文件路径，例如 `src/views/finance/rechargeType/index.vue`

如果一个任务覆盖多个页面，优先写目录，不要只写业务名。

---

## 3. 最终标准提示词

```text
请基于本仓库既有 i18n 方案，为 [模块/页面路径] 做模块内多语言接入。要求：优先复用已有 key；缺失时至少补齐 zh/en，若目标 namespace 已存在 vi/id 资源也同步补齐；确认并声明正确的 namespace；所有文案必须支持 switchLocale 无刷切换，禁止在模块顶层缓存 t() 结果；列表列、选项、label map、表单校验、message、tooltip 等文案必须放在模板、computed、render 或运行时工厂中；保持业务逻辑、状态值、查询参数、接口调用和返回处理不变。请先给出受影响文件清单和执行步骤，再开始改动；完成后汇报验证结果、风险点、手工烟测入口，以及是否同步更新了相关开发文档。
```

---

## 4. 极短版提示词

```text
基于本仓库既有 i18n 方案，为 [模块/页面路径] 做模块内多语言接入：至少补齐 zh/en；若目标 namespace 已存在 vi/id 资源，同步补齐；复用 key、声明正确 namespace、禁止顶层缓存 t()、保证 switchLocale 无刷生效、不改业务逻辑，并完成验证与文档更新。
```

适用场景：

- 你已经确认过模块边界
- 你不需要 AI 再重复解释基本规则
- 你只想快速发起标准改造任务

---

## 5. 可选追加要求

根据任务场景，可以在标准提示词后追加下面任意一条。

### 5.1 先盘点再实施

```text
先只做盘点，不要立即改代码；先输出受影响文件、风险点、建议 namespace 和预计改动范围，等我确认后再实施。
```

### 5.2 只改页面层，不动共享模块

```text
优先在页面层完成接入，除非确实无法满足 switchLocale 要求，否则不要扩大到共享模块。
```

### 5.3 强制补测试

```text
如果本次改动涉及共享 TS 配置、表格列配置或选项缓存，请补最小定向回归测试。
```

### 5.4 只允许最小改动

```text
保持改动收敛，不要顺手重构无关代码，不要改变现有业务交互和样式结构。
```

## 6. 直接示例

### 6.1 `takePackage` 模块

```text
请基于本仓库既有 i18n 方案，为 src/views/operations/takePackage 做模块内多语言接入。要求：优先复用已有 key；缺失时至少补齐 zh/en，若目标 namespace 已存在 vi/id 资源也同步补齐；确认并声明正确的 namespace；所有文案必须支持 switchLocale 无刷切换，禁止在模块顶层缓存 t() 结果；列表列、选项、label map、表单校验、message、tooltip 等文案必须放在模板、computed、render 或运行时工厂中；保持业务逻辑、状态值、查询参数、接口调用和返回处理不变。请先给出受影响文件清单和执行步骤，再开始改动；完成后汇报验证结果、风险点、手工烟测入口，以及是否同步更新了相关开发文档。
```

### 6.2 只做盘点版本

```text
请基于本仓库既有 i18n 方案，盘点 src/views/operations/takePackage 的模块内多语言接入范围。先只做盘点，不要立即改代码；先输出受影响文件、风险点、建议 namespace、是否存在顶层缓存 t() 的写法，以及建议的最小验证方案，等我确认后再实施。
```
