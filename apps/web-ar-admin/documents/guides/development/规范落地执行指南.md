# 规范落地执行指南

## 1. 目标

把当前项目已经沉淀的 `coding / architecture / api / data / ui / infrastructure / quality / reference` 文档体系，转成一条可执行的开发路径，避免“规范很多，但真实任务里不知道先看什么、怎么串起来用”。

如果你现在还没决定“该先点哪个 skill、该先看哪类文档”，先看：

- `documents/guides/development/AI技能调用一页口令卡.md`
- `documents/guides/development/AI技能调用速查手册.md`

本文档回答“一次真实开发里先看哪些规范、按什么顺序判断、如何把规则落到动作”，不新增规则正文。

## 2. 适用范围

本指南适用于：

- 新功能开发
- Bug 修复
- 重构
- 接口改造
- 多租户、多主题、多模板相关改动
- AI 与人工共同参与的开发任务

本指南主要覆盖：

- 进入任务后的阅读顺序
- 规范命中判断
- 常见任务的文档组合
- 开发前、中、后的最小动作

本指南不直接覆盖：

- 单篇规范的完整正文
- 具体实现方案设计
- 发布平台或测试平台配置

## 3. 使用前提

开始执行前，至少应先明确：

- 当前任务属于新功能、Bug、重构还是联调
- 影响的是代码风格、架构分层、接口边界、数据模型、UI 还是基础设施
- 是否命中多租户、主题、模板、宿主、共享包等高风险范围

## 4. 标准操作步骤

### 4.1 先判断任务类型

先把任务归入一类或多类：

- 编码规范问题
- 架构分层问题
- API / DTO / request 问题
- 数据模型 / 存储 / 映射问题
- UI / 样式 / 交互问题
- 基础设施 / 配置 / 宿主问题
- 测试 / 回归 / 调试问题

### 4.2 再命中对应规范组

常用映射如下：

- 命名、TS、Vue SFC、Composables：
- `standards/coding/*`
- 目录结构、状态管理、请求链、错误处理：
- `standards/architecture/*`
- 接口定义、参数、响应、mock 联调：
- `standards/api/*`
- 领域模型、持久化、缓存、字典、金额时间敏感数据：
- `standards/data/*`
- 样式、UI 结构、设计令牌：
- `standards/ui/*`
- 环境、构建、PWA、宿主、运行时装配：
- `standards/infrastructure/*`
- 调试、测试、回归、Mock：
- `standards/quality/*`

### 4.3 再补项目背景

如果任务涉及项目结构、租户、站点、模板、终端形态，应补看：

- `reference/*`

如果任务只是局部编码问题，不必一上来全扫 `reference`。

### 4.4 再进入对应 guide

当问题进入“具体怎么操作”时，再读：

- `guides/testing/*`
- `guides/debugging/*`
- `guides/integration/*`
- `guides/release/*`

规范负责“应该怎么做”，guide 负责“具体怎么操作”。

### 4.5 执行中保持最小闭环

一次真实开发至少应完成：

1. 命中相关规范
2. 做实现或修复
3. 做必要测试或调试验证
4. 做最小回归
5. 需要时补文档或更新入口

## 5. 常见场景

### 5.1 新增一个接口页面

优先看：

- `standards/architecture/*`
- `standards/api/*`
- `standards/data/*`
- 如果涉及交互，再看 `standards/ui/*`

### 5.2 修一个线上 Bug

优先看：

- `standards/quality/*`
- 命中对应代码层规范
- 如果涉及请求或数据，再补 `api/data`

### 5.3 做多租户或模板相关改动

优先看：

- `reference/*`
- `standards/infrastructure/*`
- `standards/ui/*`

### 5.4 重构一个共享模块

优先看：

- `standards/coding/*`
- `standards/architecture/*`
- `standards/quality/*`

## 6. 常见错误

- 只看一篇规范就开始做，忽略相邻层
- 一上来全量扫所有文档，导致上下文失控
- 只看 `standards`，不看 `guides`
- 只看 `guides`，不回到上位规范
- 改了实现但没做对应验证
