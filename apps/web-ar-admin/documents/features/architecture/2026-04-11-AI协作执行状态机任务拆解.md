# 3. B1 全单元输出（本次完成）

## 3.1 B1-EU1 状态机边界定义（状态/事件/门禁）

### 3.1.1 执行状态机（生命周期状态）

- discussing：需求澄清中（仅讨论/只读分析）
- planning：任务拆解中（批次与EU编排）
- ready：拆解完成，待执行授权
- executing：执行中
- stage_review：当前批次验收中
- stage_commit_ready：批次验收通过，待批次提交
- paused：人工暂停或必停触发
- blocked：门禁阻断（授权不足/校验失败）
- completed：任务完成并收口

### 3.1.2 状态转换事件

- event.scope_frozen：边界冻结 -> planning
- event.plan_frozen：批次与EU冻结 -> ready
- event.exec_authorized：收到“开始执行”等授权 -> executing
- event.eu_pass：当前EU验证通过 -> 下一EU或stage_review
- event.eu_fail：当前EU验证失败 -> blocked
- event.stage_review_pass：批次验收通过 -> stage_commit_ready
- event.stage_review_fail：批次验收失败 -> executing（新增修复EU）
- event.stage_committed：批次提交完成 -> ready（下一批次）或completed
- event.pause：人工暂停 -> paused
- event.resume：恢复执行 -> executing

### 3.1.3 门禁规则（冻结）

1. 授权门禁

- “继续”只允许流程推进/分析推进，不构成文件增删改授权。
- 进入文件修改前，必须先回报“修改文件清单+改动意图+风险”，并等待明确授权。

2. EU门禁

- 同时仅允许一个EU为 in_progress。
- EU验证失败不得进入下一EU。

3. 批次门禁

- 当前批次EU全部 completed 后，才允许进入批次验收。
- 批次验收通过后，才允许执行一次批次提交（一个批次一次commit）。

4. 并行门禁

- 仅允许同批次内并行；上限3。
- 涉及同一核心文件任务必须串行。

## 3.2 B1-EU2 数据模型与存储/恢复方案（冻结）

### 3.2.1 数据模型

- BatchState
  - batchId: string（示例：B1）
  - status: pending | in_progress | review | commit_ready | committed | failed
  - euTotal/euDone/euFailed: number
  - startedAt/updatedAt: string
- EUState
  - euId: string（示例：b1-eu2）
  - batchId: string
  - title: string
  - status: pending | in_progress | completed | cancelled
  - verification: { result: pass | fail, summary: string, evidencePaths: string[] }
  - filesChanged: string[]
- RunContext
  - currentBatchId/currentEuId/nextEuId
  - hasExecutionAuthorization: boolean
  - lastAuthorizationText: string
  - lastActionAt: string

### 3.2.2 存储与恢复策略

- 存储载体：优先本地文件快照（.local-work/session-context/），避免污染仓库。
- 快照内容：BatchState[] + EUState[] + RunContext。
- 刷新时机：EU状态变化、授权状态变化、批次验收结论写入后立即落盘。
- 恢复流程：启动先读最新快照 -> 校验结构 -> 回填“当前批次/当前EU/最近验证/下一步/授权状态”。
- 异常兜底：快照损坏时回退最近一份可用快照，并输出告警。

## 3.4 B1-EU4 执行与验收口径冻结

- 执行口径
  1. 1个EU = 1个todo项
  2. 单时刻仅1个 in_progress
  3. EU必须先验证后流转
  4. 批次内EU全通过后再批次验收
  5. 批次验收通过后执行一次批次commit
- 验收口径
  - EU验收：有验证证据、结论明确、文件清单完整
  - 批次验收：门禁规则触发正确、恢复链路可用、状态回报标准化
  - 全链路验收：支持中断恢复、授权门禁生效、违规流转被阻断
- 输出口径（每次回报必须包含）
  - 当前阶段/当前批次/当前EU/校验状态/文件清单/下一步授权语义

# 4. 每EU必填验证证据

- 执行命令
- 结果摘要
- 证据路径（日志/截图）
- 结论（pass/fail）
- 失败处置（回滚/修复后二次验证）
- 文件清单

# 5. 中断恢复机制

恢复时固定读取：

1. 当前批次
2. 当前EU
3. 最近验证结果
4. 下一步
5. 是否具备执行授权
